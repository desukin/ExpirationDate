## 技术实现细节

### 1. 保质期（Expiration）

- 使用 BaseLib `SavedSpireField<CardModel, int>`，给每张 CardModel 动态附加一个持久化计数器
- `CardPileCmd.Add` Prefix：卡进 Deck 时初始化计数器 = 5
- `Hook.AfterCombatEnd` Postfix：战后遍历 Deck，计数器 -1，归零时 `CardPileCmd.RemoveFromDeck`
- **不占用 Enchantment/Affliction 槽位**，SL 持久化

### 2. 随机附魔（Random Enchant）

- Patch `CardPileCmd.Add` Prefix（和功能1共用一个 Patch 入口）
- 21种原版 Enchantment（排除 Deprecated/Clone），随机选取
- `CanEnchant(card)` 预检查兼容性（卡牌类型、是否已有附魔等）
- 不兼容则跳过，静默失败

### 3. 全是事件（All Events）

- Patch `Hook.ModifyGeneratedMapLate` Postfix
- 遍历地图上所有节点，将 `MapPointType.Monster` 改为 `MapPointType.Unknown`
- 不影响精英和Boss房间

### 4. 敌人成长（Scaling HP）

- Patch `Hook.BeforeCombatStart` Postfix（无参数，通过 `CombatManager.Instance` 反射获取 `_state`）
- `RunManager.Instance.RunTime` 获取已运行秒数 → 转分钟
- `multiplier = 1.0 + minutes / 100.0`
- 遍历 `state.Enemies`，反射调用 `SetMaxHpInternal` / `SetCurrentHpInternal`
- 只增加不减少

### 5. 隐藏血量（Hide Enemy HP）

- 使用 `TargetMethod` 反射安全查找方法（兼容不同游戏版本）
- 优先查找 `RefreshText`，备选 `UpdateHealthDisplay`
- 反射读取 `_creature` 判断 `IsPlayer`，跳过玩家
- 隐藏 HP 条容器（`_hpForegroundContainer`）和数字标签

## 已知限制

1. **保质期计数器对已存在的卡不生效**：只有进入牌组时被 Prefix 拦截的卡才会被追踪。装 mod 后开新跑团或中途加入的卡才受影响
2. **随机附魔占用附魔槽位**：每张卡只有一个 Enchantment 槽位，随机附魔后无法再手动附魔
3. **敌人成长在 SL 后重新计算**：RunTime 是本次游戏会话的运行时间
4. **隐藏血量只隐藏数字**：血条（红/绿色进度条）仍然可见
